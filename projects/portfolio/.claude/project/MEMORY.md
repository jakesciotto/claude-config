## Design Direction

**Reference:** Supabase dashboard/marketing site aesthetic.

The portfolio should always aim for the level of visual cleanliness and polish found in Supabase's design language. Key traits to internalize:

- **Restrained color palette** -- dark backgrounds with subtle contrast, minimal use of accent color (green in Supabase's case). Avoid visual clutter.
- **Generous whitespace** -- let elements breathe. Dense information is fine, but it should never feel cramped.
- **Crisp typography** -- clear hierarchy, consistent sizing, no decorative fonts. Mono where it fits the developer aesthetic.
- **Subtle borders and separators** -- thin, low-contrast lines to define regions without drawing attention.
- **Muted UI chrome** -- buttons, cards, and navigation should feel integrated, not bolted on. No drop shadows or gradients unless they serve a purpose.
- **Data-forward layout** -- information presented clearly and directly, like a well-designed dashboard. No filler content or decorative elements that don't communicate something.
- **Consistent spacing and alignment** -- pixel-level consistency across components. Grid discipline.

When making any UI/design decision, ask: "Would this feel at home on supabase.com?"
